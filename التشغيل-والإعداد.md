# دليل الإعداد والتشغيل — نظام أم فارس

هذا الملف يلخّص حالة المشروع وما تم إضافته من تصاريح/مفاتيح وكيفية تشغيله.

## حالة المشروع
المشروع **مكتمل برمجياً** (خادم + واجهة + إعداد أندرويد). تم فحص جميع الملفات والاستيرادات والتأكد من ترابطها. الناقص كان فقط **ملفات الإعداد والمفاتيح السرية**، وقد أُضيفت.

## المتطلبات
- **Node.js نسخة 22.9 أو أحدث** (الأفضل النسخة 24). السبب: الخادم يستخدم وحدة SQLite المدمجة `node:sqlite` وميزة `--env-file`.
- npm.

## ما تمت إضافته (التصاريح والمفاتيح)
- `server/.env` — يحتوي على **مفتاح JWT سري** تم توليده، بالإضافة إلى المنفذ وإعداد CORS. هذا هو "تصريح" توقيع جلسات الدخول.
- `server/.env.example` — قالب يوضّح المتغيرات (للمشاركة بأمان).
- `client/.env` — متغير `VITE_API_BASE` (فارغ = نفس الأصل للويب).
- ربط ملف `.env` بالخادم عبر تعديل أوامر التشغيل (`--env-file-if-exists=.env`).
- تحديث `.gitignore` لحماية `server/.env` من الرفع إلى Git.

> ملاحظة مهمة: لا يحتاج هذا التطبيق أي **مفاتيح API خارجية أو اشتراكات مدفوعة** — فهو مكتفٍ ذاتياً (تسجيل دخول JWT + bcrypt + قاعدة بيانات SQLite محلية). "الاشتراكات" الوحيدة هي حزم npm المذكورة في `package.json`.

## التثبيت والتشغيل
```bash
# 1) تثبيت كل الحزم (الاشتراكات): الجذر + الخادم + الواجهة
npm run install:all

# 2) التشغيل (تطوير): الخادم + الواجهة معاً
npm run dev
```
- الواجهة: http://localhost:5173
- الـ API: http://localhost:4000

## للنشر (Production)
```bash
npm run build   # يبني الواجهة في client/dist
npm start       # الخادم يقدّم الواجهة + الـ API على المنفذ 4000
```

## بناء تطبيق أندرويد
```bash
cd client
npm run build
# اضبط VITE_API_BASE في client/.env إلى رابط الخادم العام قبل البناء للأندرويد
npx cap sync android
npx cap open android   # يفتح المشروع في Android Studio للبناء/التوقيع
```
تصريح الإنترنت (`INTERNET`) موجود مسبقاً في `AndroidManifest.xml`.

## توليد مفتاح JWT جديد (للإنتاج)
```bash
node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"
```
ثم ضع الناتج في `server/.env` مكان `JWT_SECRET`.
